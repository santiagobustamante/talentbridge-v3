# Plan de Mejoras de Frontend (UX/Producto) — TalentBridge V3

**Estado: planificado, sin implementar.** Basado en la [auditoría UX/UI del frontend](auditoria-frontend-ux.md) del 11/07/2026 (hallazgos completos, con archivo y línea) y en 14 decisiones de producto respondidas por el usuario ese mismo día. Este documento es el plan de ejecución de esa auditoría — es un trabajo nuevo y más amplio que el de [`plan-consistencia-css.md`](plan-consistencia-css.md) (que sigue completo y es la base sobre la que se construye esto).

## Contexto

La revisión consultiva encontró que el problema del frontend no es de estilo (colores/botones/cards ya están unificados) sino de **identidad fragmentada** (dos nombres de marca), **implementaciones duplicadas** (dos editores de perfil, dos dashboards candidato, dos renders del portafolio) y **manejo inconsistente de estados** (carga/vacío/error, formatos de fecha). Se sumó un hallazgo de datos: el perfil demo del propio usuario tiene ~150 habilidades cargadas del catálogo completo. Detalle completo, módulo por módulo, con cita de archivo:línea para cada hallazgo: [`auditoria-frontend-ux.md`](auditoria-frontend-ux.md).

## Restricciones del proyecto

- Desarrollo 100% local hasta nueva indicación (ver memoria del proyecto).
- Antes de borrar cualquier dato real (ver Fase 0), se pide confirmación explícita puntual — una respuesta de opción múltiple durante la ronda de preguntas no cuenta como esa confirmación.
- Un solo commit al final de todo el plan, salvo que el usuario pida lo contrario (mismo criterio que el plan de CSS).

## Decisiones de producto ya tomadas (11/07/2026)

| Tema | Decisión |
|---|---|
| Marca | Unificar todo a **TalentBridge** |
| Dirección visual | Red profesional tipo **LinkedIn** |
| Paleta de color | **Opción B — Teal profesional** (`#0f766e`), elegida viendo 3 propuestas comparativas |

| Densidad de información | Mantener la actual |
| Módulos clave | Todos — portafolio/perfil público, trabajos/postulaciones, panel empresa, chat/Joaquín |
